PATCH ADMIN SEM SUPABASE AUTH

O console mostrou CORS exatamente na rota /auth/v1/token. Este patch evita o Supabase Auth no painel admin.

Como usar:
1. Suba admin.html, config.js e assets/admin.js para o GitHub, substituindo os atuais.
2. No config.js, cole sua chave anon public que começa com eyJ no lugar indicado.
3. Troque window.ADMIN_PASSWORD para uma senha sua.
4. No Supabase > SQL Editor, rode patch_policies_admin_sem_auth.sql.
5. Entre no admin usando apenas a senha definida em config.js.

Observação:
Esta é uma solução prática para GitHub Pages sem backend. A senha local fica no código publicado, então não é segurança forte.
